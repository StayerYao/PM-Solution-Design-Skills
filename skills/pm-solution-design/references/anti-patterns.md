# 常见合理化借口

| 借口 | 现实 |
|------|------|
| "需求太简单，不用深究问题" | NEVER 跳过深究，简单需求最易因假设错误返工 |
| "用户已经说了要什么" | MUST 追问问题，用户说的通常是方案，不是问题 |
| "异常情况以后再说" | MUST 定义异常，未定义 = 生产事故 |
| "硬编码先上线，以后再抽象" | NEVER 硬编码先上线，以后永远不会来 |
| "门禁太重了，先跳过" | NEVER 跳过门禁，跳过省的时间会在实现阶段加倍还回 |
| "这个主观词大家都懂" | MUST 替换主观词，"大家都懂" = 每个人理解不同 |
| "功能变了用户自己会适应" | MUST 声明用户影响，严禁静默处理 |
| "接口设计我先顺手写了" | MUST NOT 写接口设计，超出 PRD 边界 |
| "加载中不用特别处理" | MUST 定义等待态，没有等待态 = 用户以为卡死 |
| "这种错误一般不会发生" | MUST 定义异常，未定义 = 生产事故 |
| "这个跟那个差不多，处理方式显而易见" | MUST 显式声明，"显而易见" = 两个人理解不同 |
| "先这样，体验细节后面再打磨" | NEVER 延后品质，设计阶段品质上限锁死后无法突破 |
