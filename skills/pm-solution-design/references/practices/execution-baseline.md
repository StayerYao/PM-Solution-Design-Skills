# Execution Baseline

执行基线用于 G3 执行契约门禁。目标是避免 PRD 看似完整，但研发和测试仍不知道关键状态如何处理。

## 基线约束

异步操作必须定义等待态：提交、导入、导出、生成、审核、支付、同步、批处理等操作，都必须说明处理中用户看到什么、是否可取消、是否可重试。

不可逆操作必须定义确认机制：删除、发布、撤回、扣费、扣积分、权限变更、批量修改等操作，必须说明确认步骤、风险提示和误操作后的处理方式。

错误响应必须包含可执行下一步：错误提示必须让用户理解失败原因，并知道下一步可以重试、修改输入、联系谁、返回哪里或等待多久。

空态必须有业务语义：无数据、无权限、无搜索结果不能只显示空白，必须说明原因和引导。

重复操作必须有处理规则：重复提交、重复点击、并发编辑或重复导入必须定义系统行为。

## 验收写法

合格：用户提交导入文件后，系统展示导入处理中状态；处理失败时展示失败原因和重新上传入口。

不合格：系统快速处理导入，失败时提示错误。

## 失败处理

主观词、格式或优先级缺失时自行修订。验收口径依赖业务定义，或错误下一步无法判断时，必须停问用户。
